# Chapter 3 - Machine-Level Representation of Programs

- why learn assembly? - to understand the code generated by the compiler for code optimization and security

## 3.1 - A Historical Perspective
- Intel has been producing cpus since the 1970's, and each designed with backward compatibility, which leads to some strange artifacts in the instruction set
- x86-64 is an extension of IA32 (Intel Architecture 32-bit)

## 3.2 - Program Encodings
### 3.2.2 - Code Examples

`gcc -Og -S filename.c` to generate assembly <br>
`gcc -Og -c filename.c` to compile and assemble <br>
disassembler generates assembly code from binary <br> 
`example: objdump -d filename.o` <br>

```
multstore:
.LFB0:
	.cfi_startproc
	pushq	%rbx
	.cfi_def_cfa_offset 16
	.cfi_offset 3, -16
	movq	%rdx, %rbx
	call	mult2@PLT
	movq	%rax, (%rbx)
	popq	%rbx
	.cfi_def_cfa_offset 8
	ret
	.cfi_endproc
```

lines beginning with **'.'** are directives to guide assembler and linker <br>

## 3.3 - Data Formats
| C Declaration | Intel Data Type        | Assembly Suffix | Size (bytes) |
|---------------|------------------------|-----------------|--------------|
| char          | Byte                   | b               | 1            |
| short         | Word                   | w               | 2            |
| int           | Double word            | l               | 4            |
| long          | Quad word              | q               | 8            |
| char *        | Quad word              | q               | 8            |
| float         | Single precision       | s               | 4            |
| double        | Double precision       | l               | 8            |

## 3.4 - Accessing Information
- an x86-64 cpu has 16 generate purpose registers storing 64 bit values, beginning with %r
- each register has a 64 bit, 32 bit, 16 bit, and 8 bit register, but they all share the same memory
|---------------------- rax (64 bits) ----------------------| <br>
|----------- eax (32 bits) -----------| <br>
|------- ax (16 bits) ------| <br>
|--- al (8 bits) ---| <br>

### 3.4.1 - Operand Specifiers
- Most assembly instructions have one or more operands
- Operand Types
- **immediate** - constant values (begin with $)
- **register** - contents of a register
- **memory** - access a memory address according to a computed address
